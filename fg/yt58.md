# Football Operations Plan

## Objective
Build a dedicated super-admin football operations page and end-to-end workflow comparable to the cricket desk, so football fixtures, live updates, platform odds generation, provider reference odds, review, publishing, and monitoring can be handled from one place.

## Current Audit

### What already exists
1. Generic provider system exists.
- Super admin can add providers and feeds.
- Football-capable providers currently present in code:
  - `api_sports`
  - `allsports`

2. Generic competition feed system exists.
- Feeds support:
  - sport
  - provider
  - competition key
  - import mode
  - live sync
  - provider odds import flag
  - platform odds generation flag

3. Generic match import and admin match workspace exists.
- Imported matches can be viewed in `/admin/matches`.
- Match detail page exists.
- Odds workspace exists at `/admin/matches/[id]/odds`.
- AI tools already exist:
  - generate odds
  - regenerate
  - rewrite with comment
  - orchestrate
  - publish
  - unpublish

4. Football AI and settlement foundations exist.
- Football odds strategy exists.
- Football in-play settlement/snapshot logic exists.
- Football market rules exist.
- Sport market config supports football.

### What does not exist yet
1. No dedicated football page in super-admin sidebar.
- Current sidebar has `Cricket`, but no `Football`.

2. No football discovery/catalog page.
- There is no football equivalent of the cricket operations desk.
- No league browser.
- No football-specific grouping by league/competition.

3. No football-specific feed helper flow.
- Users must use generic feeds manually.
- No fast workflow for EPL, UCL, La Liga, etc.

4. No dedicated football live trading desk.
- Current odds workflow is only through generic match pages.
- No football page with tabs like:
  - Competitions
  - Upcoming
  - Live
  - Draft Odds
  - Published Odds
  - Closed

5. Provider reference odds are not fully implemented for current football providers.
- `api_sports` adapter currently returns `{:error, :provider_odds_not_supported}`.
- `allsports` adapter currently returns `{:error, :provider_odds_not_supported}`.
- So provider-odds comparison/import is not yet real for football using those adapters.

6. No football automation layer like cricket feed automation.
- Cricket feed config supports auto prematch and in-play draft generation.
- Football does not yet have a dedicated football desk driving that workflow.

## Required End State

A super admin should be able to do all football operations from one page:
1. Add or activate football provider.
2. Discover or create football league feeds.
3. Import upcoming fixtures by league/season.
4. Refresh live football matches.
5. Review football matches grouped by competition.
6. Generate prematch odds per match.
7. Generate in-play odds per live match.
8. Run AI orchestration and rewrite with admin comment.
9. Review draft odds before publish.
10. Publish or unpublish odds.
11. Optionally compare against provider reference odds where supported.
12. Monitor sync failures, feed metrics, and automation state.

## Screen Model

### New super-admin page
`/admin/football`

### Top tabs
1. `Competitions`
2. `Upcoming`
3. `Live`
4. `Draft Odds`
5. `Published Odds`
6. `Closed`
7. `Settled`
8. `Cancelled`

### On-page workflow
- `Competitions`
  - provider picker
  - league discovery or manual feed creation
  - feed cards
  - import upcoming
  - refresh live
  - enable automation

- `Upcoming`
  - grouped by league/competition
  - per match:
    - generate odds
    - orchestrate
    - rewrite
    - open odds desk
    - match detail

- `Live`
  - grouped by league/competition
  - live score/status visible
  - in-play automation status
  - direct trading actions

- `Draft Odds`
  - show only matches with draft platform odds
  - quick review and publish flow

- `Published Odds`
  - show only matches with published platform odds
  - quick review and unpublish flow

## Build Phases

### Phase 1: Audit-safe football operations shell
Goal: create the page and unify existing generic capabilities under a football-only surface.

Tasks:
1. Add `/admin/football` page.
2. Add `Football` to super-admin sidebar.
3. Reuse the cricket-page structural model, but scoped for football.
4. Read football feeds only.
5. Read football matches only.
6. Group by competition/league.
7. Reuse generic match actions and odds workspace links.

Deliverable:
- Football page exists and acts as the football control center, even before provider discovery enhancements.

### Phase 2: Football feed creation and league workflow
Goal: make feed creation practical for football.

Tasks:
1. Decide primary football provider path.
   - likely `api_sports` first
   - `allsports` optional fallback
2. Add football-specific feed form presets.
3. Add football helper copy and reduced form friction.
4. Add league/season/competition organization rules.
5. Add duplicate protection for football feeds similar to cricket.

Deliverable:
- Super admin can create football feeds from a football page without dropping into generic provider workflows.

### Phase 3: Football competition discovery
Goal: let super admin browse football leagues from provider instead of manually guessing IDs.

Tasks:
1. Add provider-backed football discovery endpoint.
2. Normalize competition payload.
3. Cache discovery results.
4. Group by region and competition tier.
5. One-click `Create Feed + Import` from discovered football competitions.

If provider discovery is weak:
- add manual assist flow with league IDs and season IDs.

Deliverable:
- Super admin can browse leagues like EPL, La Liga, UCL, Serie A from inside `/admin/football`.

### Phase 4: Football match board
Goal: make football upcoming/live management practical.

Tasks:
1. Football match cards with:
   - home/away names
   - logos when present
   - competition
   - kickoff time
   - score/status for live
2. League grouping in all major tabs.
3. Quick actions on each card.
4. Feed/league badges and sync status.

Deliverable:
- Upcoming and live football tabs become a real trading board rather than a raw imported list.

### Phase 5: Football automation controls
Goal: match cricket automation maturity.

Tasks:
1. Extend feed config handling for football automation.
2. Prematch automation toggle.
3. In-play automation toggle.
4. Prematch window settings.
5. In-play interval settings.
6. Max runs per match.
7. Match-level latest automation status.

Deliverable:
- Football feeds can auto-generate draft prematch and in-play odds safely.

### Phase 6: Football provider reference odds support
Goal: enable optional provider-odds comparison/import where real provider support exists.

Tasks:
1. Audit `api_sports` odds availability and exact endpoints.
2. Audit `allsports` odds availability and exact endpoints.
3. Implement `fetch_odds_for_match/2` only if the provider actually supports it in the current plan/endpoint.
4. Normalize provider reference odds.
5. Show them in the football desk and generic odds workspace.
6. Allow import as draft/provider reference when valid.

Important:
- This phase is conditional.
- Do not fake provider odds if the provider does not support them.

Deliverable:
- Football provider odds become real and optional where available, otherwise clearly marked unsupported.

### Phase 7: Football-specific odds UX
Goal: make football odds handling feel native rather than generic.

Tasks:
1. Football market label mapping:
   - match winner
   - double chance
   - both teams to score
   - goal lines
   - in-play goal markets
2. Football market ordering.
3. Better football live context chips.
4. Cleaner football draft preview blocks.

Deliverable:
- Football operators can review odds in a football-native way.

### Phase 8: Monitoring and operator polish
Goal: make the page production-operable.

Tasks:
1. Feed metrics per competition.
2. Sync failure callouts.
3. Last fixture sync / last live sync.
4. Last provider-odds fetch/import where supported.
5. Empty states and warnings.
6. Mobile-safe layout.

Deliverable:
- Football desk is operationally safe and understandable.

## Technical Constraints
1. Do not break cricket desk.
2. Do not regress generic admin match/odds pages.
3. Do not claim provider odds support unless endpoint support is real.
4. Keep mobile usable.
5. Keep loading light for low-end devices.
6. Prefer CSS transitions only, no heavy JS animation.

- Football is not yet managed from a dedicated super-admin page like cricket.
- Provider reference odds are not fully implemented for the currently wired football adapters.
- A full football operations desk still needs to be built.
